# Documentation Style Guide

## Purpose
Define uniform documentation structure for AI-HPP public materials.

## Scope
Applies to root-level governance files and documents under `docs/` and `agent_safety/`.

## Structure Requirement
Each normative section should include:
- Purpose
- Scope (if applicable)
- Requirements
- Implications (optional)

## Rules
- No duplicated principles across files
- No motivational or promotional language
- Clear terminology and stable naming
- Minimal redundancy
- Consistent headings
