# AI-HPP Compliance Checklist

## Purpose
Provide a compact operational checklist for public AI-HPP declarations.

## Scope
Applies to public self-declared compliance at Core, Agent, and Enterprise levels.

## Requirements
### Basics
- Public system link provided
- Compliance declaration published
- Compliance level clearly stated

### Core
- Decision logging implemented
- Injection mitigation documented
- Structured output enforced
- Refusal policy active

### Agent
- Execution separated from reasoning
- Tools classified and limited
- Memory controlled
- No self-escalation
- Failure events logged

### Enterprise
- Economic limits enforced
- Human veto defined
- Monitoring described

## Implications
Failure to satisfy listed items should block badge claims at the corresponding level.
